Lab2 System Calls
2022-08-17

在本实验中，需要向xv6中增加一些新的系统调用

Lab2前置要求

  • 阅读xv6 book的第2章、第4章4.3以及4.4
  • 系统调用的用户空间代码在user/user.h和user/usys.pl中
  • 内核空间代码是kernel/syscall.h， kernel/syscall.c
  • 与进程相关的代码是kernel/proc.h和kernel/proc.c

System call tracing

  • 将trace的函数原型加入user.h

  • 将存根加入usys.pl

  • 根据第二条hint，在syscall.h中加入系统调用号(macro)

  • 在kernel/sysproc.c中添加一个sys_trace函数

  • 在proc结构体中加入mask对应上述函数的mask变量

  • 对syscall中的syscall函数进行修改，使其输出trace跟踪的结果

  • 同时需要在syscall.c中增加对于函数的声明

  • 在fork函数中，将父进程的mask值复制给子进程

Sysinfo

在最开始的user.h以及usys.pl文件中的操作与前面的trace非常类似，在此不做赘述

  • 由于sysinfo需要统计当前系统的相关信息 ===> 剩余可用空间/已用线程数 ===> 需要对应的函数

  • 在kernel/sysproc.c中添加一个sys_trace函数

  • 和sys_trace相同，需要在syscall.c中增加函数的声明

心得

  • hint很重要，其中会给出完成整个实验的需要完成的事(几乎每一步都有涉及到)

  • 对于各种.c文件中新增的函数，函数声明应当放在kernel/defs.h中；一度因为这个问题迷惑很久

  • 完成本实验的过程，更多是对于文档和提供的源代码，对于xv6源代码的阅读对于个人编程有着极大的帮助(学习编程风格、范式/对于函数的使用===>更好地完成实验)
